# アイデアの回し方 — 出す・貯める・標本化する

モーション図鑑は「完成した標本」だけでなく、**思いつきを貯める苗床**を持っています。
台帳は [`src/ideas.ts`](src/ideas.ts) の1ファイル。ここに追記すれば、図鑑下部の
「アイデアの苗床」セクションに自動で並びます。

## ライフサイクル

```
seed（種） ──育てる──▶ sprout（育成中） ──実装──▶ captured（標本化済み）
 一行の思いつき          動きの設計を詰める        specimens/ に住む
```

- **seed は質より量**。動くかどうか・実装できるかは考えず、一行で書き捨てる
- **sprout に上げる基準**：「何が・どんな緩急で・何を伝えるか」の3点が言えること
- **captured にする手順**は下記「標本化」参照

## 1. 出す（案出しのレシピ）

行き詰まったら、掛け算で機械的に生成する：

| 軸A：UIの部品 | × | 軸B：動きの性格 |
| --- | --- | --- |
| ゲージ・カウンター・ボタン・カード・リスト・アイコン・帯・遷移 | | 跳ねる・潰れる・溶ける・震える・じらす・崩れる・吸い付く・鼓動する |

さらに**ゲームの感情**を掛けると幅が出る：
「報酬（嬉しい）」「被弾（痛い）」「待ち時間（そわそわ）」「緊張（ハラハラ）」「祝福（レベルアップ）」「エラー（ダメ）」

例：`ゲージ × 溶ける × 被弾` → HPバーの遅延ゴースト（No.13）

## 2. 貯める（台帳への追記）

`src/ideas.ts` の `ideas` 配列に1エントリ足すだけ：

```ts
{
  id: 'boss-banner',            // ユニークなslug。標本化したらフォルダ名になる
  title: 'ボス登場の帯',
  motion: '黒い帯が左右からシャキーンと交差し、名前が一拍遅れてドンと据わる',
  tags: ['ゲーム', '演出', '緊張感'],
  status: 'seed',
},
```

`motion` は「何が・どんな緩急で・何を伝えるか」を一行で。
オノマトペ（ぷるん・ドン・しゅるっ）で書くと、あとで実装するとき緩急の設計図になる。

## 3. 標本化（seed → 図鑑へ）

1. `pnpm new <id>` で `src/specimens/<id>/`（index.tsx + style.css）の雛形を生成
2. 動きを実装する。ルールは既存標本と同じ：
   - 依存ライブラリなしのCSSのみ。クラスは `mz-<id>-*` プレフィックス
   - 標本フォルダ単体で自己完結（コピーすれば他プロジェクトで動く）
   - 基本イージングは `cubic-bezier(0.34, 1.56, 0.64, 1)`（ぷるん）
   - `prefers-reduced-motion` はグローバルCSSが面倒を見る
3. `src/registry.ts` の `specimens` 配列にメタデータ（No・名前・生態）を追記
4. `src/ideas.ts` の該当エントリを `status: 'captured'` + `specimenId: '<id>'` に更新

これで図鑑と苗床の両方に反映され、「どの種がもう育ったか」が台帳に残ります。

## メンテナンスの目安

- 種が尽きてきたら、上のレシピで10個ばら撒く（1案1分でいい）
- 標本化は「触って気持ちいいか」で選ぶ。人気の種から育てる必要はない
- ボツにした種も消さなくていい。安い在庫は他の案の掛け算相手になる
